# Walkthrough video ‚Äî regenerated every build

A self-shooting product tour of marc.portal: **Home ‚Üí Intake ‚Üí Login ‚Üí magic
link ‚Üí /me ‚Üí Session (Marc's reply)**, plus a **short clip per page**. It
re-screenshots the *live* app every build, so the video never drifts from the
real product.

```
public/walkthrough/
  timeline.js     ‚Üê creative layer (shots, camera, cursor, captions). Hand-edited, stable.
  player.html     ‚Üê frame-driven player. Aims camera/cursor at named anchors.
  manifest.json   ‚Üê GENERATED by capture.mjs (frames + anchor boxes from the live build)
  frames/*.png    ‚Üê GENERATED (full-page screenshots @1620w)
  video/*.webm    ‚Üê GENERATED (full + one per page) by render-video.mjs
scripts/walkthrough/
  capture.mjs      ‚Üê screenshots live routes, records each anchor's box
  render-video.mjs ‚Üê records player.html to webm (full + per-page shorts)
```

## How it stays in sync
`timeline.js` never names pixels ‚Äî only **anchors** (`cta`, `field`, `submit`,
`card`, `marc`, ‚Ä¶). `capture.mjs` finds those elements in the live DOM and writes
their real bounding boxes to `manifest.json`. The player reads the boxes and
points the camera/cursor at them. Move the layout, change the copy, restyle ‚Äî
the shot still lands. You only edit `timeline.js` to *re-cut* (new pacing,
captions, scenes).

If the DOM structure changes, the only thing to update is the `CONFIG` selector
block at the top of `capture.mjs`.

## Run it
```bash
npm i -D playwright wait-on && npx playwright install chromium
npm run build && npm run preview          # serves built app + player at /walkthrough/player.html

# 1) refresh frames + anchor boxes from the live build
BASE_URL=http://localhost:4173 \
MP_SESSION_COOKIE="<session-cookie-name>=<value>" \
MP_SESSION_ID="<a-session-that-has-a-marc-reply>" \
  node scripts/walkthrough/capture.mjs       # or: npm run walkthrough:capture

# 2) export videos (full + one short per page)
BASE_URL=http://localhost:4173 node scripts/walkthrough/render-video.mjs   # or: npm run walkthrough:video

# both in one shot once the env vars are exported:
BASE_URL=http://localhost:4173 npm run walkthrough
```

**`vite preview` serves only the static SPA ‚Äî no `/api/*` backend.** The public
routes (home, intake, login, magic) shoot fine against it; `/me` and `/session`
need a server that also runs the Pages Functions + D1. Point `BASE_URL` at the
backend e2e harness (`http://localhost:8788`) for those ‚Äî see *Auth + seed*
below. Without a cookie, `capture.mjs` simply skips the two authed routes and
shoots the rest.

### Preview interactively (no video)
Open `http://localhost:4173/walkthrough/player.html` ‚Äî loops the full cut.
- `?scene=intake` ‚Üí just that page as a short (`home|intake|login|magic|me|session`)
- `?lang=en` ‚Üí English captions + endcard (default `fr`; both live in `timeline.js`)
- `?noend=1` ‚Üí drop the wordmark card ¬∑ `?t=4.2` ‚Üí start at a time

### Embedded on /carte
`src/components/Map/WalkthroughFilm.tsx` iframes this player as a coda under the
atlas (FR `/carte`, EN `/en/map`), with chapter chips for the full cut + each
scene. It passes `?lang` so each language renders its own captions live ‚Äî no
per-language video file. Click-to-play: the iframe (and its motion) only mounts
on click, which defers the load and respects `prefers-reduced-motion`.

This is why `manifest.json` + `frames/` are **committed** (see `.gitignore`):
the embedded player needs them at runtime in prod. The player is CSP-clean
(external `player.js`, self-hosted `/fonts/webfonts.css` ‚Äî no Google Fonts, Loi
25) so it loads under the production `_headers` CSP. Refresh the committed
frames with `npm run walkthrough:capture` and re-commit, same discipline as the
e2e visual baselines. **Changing the embed or the map layout shifts the `/carte`
screenshot ‚Äî regenerate the visual baselines on the PR** (`gh workflow run
e2e-snapshots.yml`).

### Auth + seed (for /me and /session)
`/me` and `/session/:id` are behind the magic-link session. `capture.mjs` wants
a valid session cookie via `MP_SESSION_COOKIE` and a session that already has a
reply from Marc via `MP_SESSION_ID` (so the `session__reply` frame shows the
thread; `session__pending` is derived by hiding the reply).

`scripts/walkthrough/seed.mjs` produces both ‚Äî it uses the **same fixture path
as the backend e2e suite** (`e2e/backend/helpers`): the e2e `SESSION_SECRET`,
the ephemeral `.wrangler-e2e` D1, and the exact HMAC cookie format
`functions/_lib/auth.ts` verifies.

```bash
# one-time: create + migrate the e2e D1 the seed and harness share
npm run e2e:backend:setup            # or: npx wrangler d1 migrations apply marc-portal-db --local --persist-to=.wrangler-e2e

# seed a session with a Marc reply + mint the cookie; prints the two exports
node scripts/walkthrough/seed.mjs
#   MP_SESSION_COOKIE='mp_session=‚Ä¶'
#   MP_SESSION_ID='sess_walkthrough'
```

Then run `capture.mjs` with `BASE_URL` pointed at a server bound to that **same
D1 and `SESSION_SECRET`** (the e2e harness on `:8788`), or the cookie is
rejected and the session 404s. Overrides: `SESSION_SECRET` (sign with a
different key, e.g. your `.dev.vars` value to target a normal `wrangler pages
dev` against `.wrangler`), `MP_SEED_EMAIL`, `MP_SEED_SESSION_ID`,
`MP_PERSIST_DIR`. `--no-db` mints the cookie only.

## CI (GitHub Actions sketch)
To shoot **all six routes** (including `/me` + `/session`) the capture must run
against the Functions backend, not `vite preview`. Reuse the backend e2e harness
so the cookie/secret/D1 line up, and seed with `seed.mjs`:

```yaml
- run: npm ci && npx playwright install --with-deps chromium
- run: npm run build
# bring up the same backend the e2e suite uses: swaps in e2e wrangler.toml,
# wipes + migrates the ephemeral .wrangler-e2e D1 (so seed AFTER this step).
- run: npm run e2e:backend:setup
- run: node scripts/walkthrough/seed.mjs            # seeds the session, prints the exports
# start the harness server (wrangler pages dev on :8788, e2e bindings) ‚Äî same
# webServer command as playwright.backend.config.ts ‚Äî then wait for it:
- run: <wrangler pages dev ‚Ä¶ --port 8788> & npx wait-on http://localhost:8788
- env:
    BASE_URL: http://localhost:8788
    SESSION_SECRET: e2e_session_secret_at_least_32_chars_long_padding
    MP_SESSION_COOKIE: mp_session=‚Ä¶   # from seed.mjs output
    MP_SESSION_ID: sess_walkthrough
  run: npm run walkthrough
- uses: actions/upload-artifact@v4
  with: { name: walkthrough, path: public/walkthrough/video }
```

For a public-routes-only clip, the simpler `npm run preview` on `:4173` with no
cookie is enough ‚Äî `capture.mjs` skips the two authed routes.

`manifest.json` + `frames/` + `video/` are build artifacts ‚Äî gitignore them or
commit refreshed copies, your call.
